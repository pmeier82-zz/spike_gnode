 Dataset and  Datafile

What are these? A dataset consists of one or more datafiles, each a self-contained piece of data from live extracellular recordings or data simulations. Users can download and contribute datasets.

What are they good for? Datasets provide a way to distribute standard benchmark datasets to interested users.

What to do with them? After download, users apply spike sorting algorithms to the datasets. Spike sorting results submitted back to the service will result in an analysis.

Can users contribute benchmark datasets? Yes! Although there are initial benchmark datasets provided to jump start the service, we are asking for interested parties to create and upload benchmark datasets. If you have any questions or need help, we are happy to help and guide you through the process.

 Analyses and  Modules

What are these? An analysis is the report about spike sorting results submitted back to the service. They consist of a set of module outputs that form up a comprehensive performance report of the submission.

What about modules? The performance evaluation is a modular process. Each module provides a piece of the puzzle forming up a comprehensive performance report. Users do not directly interact with modules.

Will others see my analyses? Unless specifically disclosed for public inspection, analyses will be only visible to the submitting user. However, we strongly encourage disclosure, in order to build public summary statistics.

Can users contribute modules? Yes! However, the incorporation of a new analysis components into the service is a delicate step that we cannot automate at this point in time. If you want to motivate new analysis components, please contact us with a paper reference and a sketch of the results presentation you would like to see.
